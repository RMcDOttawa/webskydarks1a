TheSkyX Server Setup

What "Server"?

In the scenario we are supporting with this program, WebSkyDarks does not control your camera - TheSkyX Professional does. TheSkyX Pro must be running on your network and when we say "Server" we are referring to establishing the connection from WebSkyDarks to the running instance of TheSkyX Pro.

To make TheSkyX Pro act like a server, you need to turn on the feature that causes it to listen for commands on TCP.

In TheSkyX Pro, select "TCP Server" from the Tools menu:

Then, in the "TCP Server" window that appears, make sure the "Listen for Connections" checkbox is on:

How to Connect to the Server

Now we need to tell WebSkyDarks where the TheSkyX server is running: which computer on the network, and which port on that computer.

The computer on which TheSkyX is running can be the same computer where WebSkyDarks is running, or it can be a different computer on the same local network (e.g., a different computer on your home WiFi network).

TheSkyX and WebSkyDarks On the Same Computer

If you are running TheSkyX and WebSkyDarks on the same computer, then you can probably use "localhost" as the server address, or you can use the IP address "127.0.0.1" (a special IP address, defined by the standard, that always refers to "this computer, right here"). You can also use the network domain name of your current computer, or its real IP address, if you know them.

In general it's a good idea to avoid using real IP addresses, in case you are on a network that uses DHCP to allocate them at start-up time. In such a network, the IP address might be different next time. On the other hand, if you are using fixed IP addresses on your network, it is safe to specify the IP address.

TheSkyX and WebSkyDarks On a Different Computer

If TheSKyX is running on a different computer, enter either the local domain name for that computer, or its IP address. Inside a private network, local domain names often have the form "name.local", where "name" is specified as the network name of the computer. This name is found in the Sharing settings on a Mac, or in the Computer Properties on Windows. You probably need to add ".local" to make the name resolve on your network.

Note that network domain names can't contain spaces. So, even though your Mac allows you to give your computer a sharing name with spaces in it, don't.

In the example shown here, I am running WebSkyDarks on my desktop computer, and TheSkyX on the computer in my observatory, whose network name is EWHO3, so I enter "ewho3.local" as the address. Domain names are not case-sensitive.

Port Number

TheSkyX listens on port 3040, so fill that value in for the port number. I've left it as an editable field because I wouldn't be surprised if there is an obscure option buried somewhere deep in TheSkyX that would allow it to listen on a different port. But 3040 is almost certainly the right value.

Test Connection

With TheSkyX running on its host computer, you can use the "Test Connection" button to get a quick confirmation that WebSkyDarks is able to establish a connection using the server information you have provided.

Note that you can also test the complete operation of this setup without wear and tear on your camera, by configuring TheSkyX to use its built-in "Camera Simulator". This will allow you to rehearse using WebSkyDarks to control TheSkyX without doing actual image acquisition.
